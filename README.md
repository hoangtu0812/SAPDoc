# SAPDoc

Chào mừng bạn đến với **SAPDoc**! Đây là kho lưu trữ (repository) chuyên dụng để tổng hợp và chia sẻ các tài liệu học tập, nghiên cứu về lập trình **ABAP** cũng như các **Nghiệp vụ hệ thống SAP**.

## 🎯 Mục tiêu
- Cung cấp nguồn tài liệu học tập chất lượng, có hệ thống về ngôn ngữ lập trình ABAP.
- Lưu trữ các kiến thức thực tế về cấu hình và vận hành các phân hệ (Module) trong hệ thống SAP ERP (như FI, CO, MM, SD, HCM, v.v.).
- Trở thành sổ tay tra cứu nhanh chóng và tiện lợi cho các lập trình viên và chuyên viên nghiệp vụ SAP.

## 📚 Nội dung chính

Kho lưu trữ này được chia thành 2 phần chính:

### 1. Lập trình ABAP
- Kiến thức cơ bản về ngôn ngữ ABAP (Data Dictionary, Internal Tables, Open SQL...).
- ABAP Objects (Lập trình hướng đối tượng trong ABAP).
- Các kỹ thuật báo cáo (ALV Reports).
- BDC (Batch Data Communication), BAPI, IDoc, RFC.
- Cải tiến hệ thống (Enhancements, BAdIs, User Exits).
- Các chủ đề nâng cao (OData, Fiori, HANA...).

### 2. Nghiệp vụ hệ thống SAP (SAP Modules)
- Tài liệu quy trình nghiệp vụ (Business Processes) tiêu chuẩn của các phân hệ.
- Hướng dẫn cấu hình (Configuration Guide) trong SPRO.
- Luồng dữ liệu và tích hợp giữa các phân hệ.
- Các Transaction Code (T-Code) phổ biến và cách sử dụng.

## 📂 Cấu trúc thư mục (Gợi ý)
Bạn có thể tổ chức tài liệu theo cấu trúc sau để dễ dàng quản lý (hãy tạo các thư mục này nếu chưa có):
```text
SAPDoc/
├── ABAP/                  # Tài liệu và mã nguồn tham khảo về ABAP
│   ├── Basic/
│   ├── ALV_Reports/
│   ├── Enhancements/
│   └── Integrations/
├── Modules/               # Tài liệu nghiệp vụ các phân hệ
│   ├── MM_Materials_Management/
│   ├── SD_Sales_and_Distribution/
│   ├── FI_Financial_Accounting/
│   └── ...
├── Projects/              # Tài liệu các dự án thực tế (nếu có)
└── References/            # Các sách, bài viết, link hữu ích tham khảo
```

## 🤝 Hướng dẫn đóng góp
Nếu bạn muốn đóng góp tài liệu vào kho lưu trữ này:
1. Clone repository này về máy tính của bạn.
2. Thêm hoặc cập nhật tài liệu vào đúng thư mục tương ứng.
3. Commit các thay đổi với thông điệp (commit message) rõ ràng.
4. Push lên repository để lưu trữ.
